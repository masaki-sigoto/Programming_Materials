# AGENTS.md - エージェント構成

## ロール定義

### PM（メインエージェント）
- **役割**: タスク管理・方針決定・ユーザーへの報告
- **原則**: 実装作業は行わず、サブエージェントに委任する
- コンテキストウィンドウの節約を最優先する

### Impl（サブエージェント: 実装担当）
- **役割**: コード実装・ファイル編集
- **原則**: Plans.mdのタスクに基づいて作業する
- 作業完了後はPMに結果を報告する

### Reviewer（サブエージェント: レビュー担当）
- **役割**: コードレビュー・品質チェック
- **原則**: push前にCodexレビューを実施する
- セキュリティ・パフォーマンス・アクセシビリティの観点でチェック

### Explorer（サブエージェント: 調査担当）
- **役割**: コードベース調査・情報収集
- **原則**: 調査結果を簡潔にまとめてPMに報告する

## ワークフロー

```
1. PM: ユーザーの要望を理解し、タスクを分解
2. PM: Plans.mdにタスクを記録
3. PM → Impl: 実装を委任
4. Impl: 実装完了後、PMに報告
5. PM → Reviewer: push前レビューを依頼（Codex使用）
6. Reviewer: レビュー結果をPMに報告
7. PM → Impl: レビュー指摘の修正を依頼
8. PM: 修正確認後、commit & push
```

## Git運用ルール
- 作業ごとに細かくcommitする（1論理変更 = 1コミット）
- push前にはCodexレビューを必ず実施する
- レビュー指摘を修正してからpushする
