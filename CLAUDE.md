# CLAUDE.md - プロジェクトルール

## プロジェクト概要
- **名称**: 小学生向けプログラミング教育コンテンツサイト
- **目的**: 保護者向けに、プログラミング教育の重要性とスクールの価値を訴求するランディングページ
- **ターゲット**: 小学生の保護者（特に「スクールに通わせるべきか迷っている」層）
- **技術スタック**: HTML5 / CSS3 / Vanilla JS（フレームワーク不使用）
- **デプロイ**: GitHub Actions → WordPress REST API 自動デプロイ

## 開発ルール

### Git運用
- **作業ごとに細かくcommitすること** — 1つの論理的変更につき1コミット。大きな変更を1コミットにまとめない
- **push前にはCodexによるレビューを実施し、指摘事項を修正してからpushすること**
- コミットメッセージは日本語で、変更内容が明確にわかるように記述する

### 作業スタイル
- **基本的にサブエージェントに作業を委任し、メインエージェントはPMとして振る舞うこと** — コンテキストウィンドウの節約を最優先
- 調査・実装・レビューはそれぞれサブエージェントに委任する
- メインエージェントはタスク管理・方針決定・ユーザーへの報告に集中する

### コード品質
- セマンティックなHTML構造を維持する
- CSSはインラインスタイル(`style`属性)の多用を避け、クラスベースのスタイリングを推奨する
- レスポンシブデザイン（モバイルファースト）を意識する
- 日本語コンテンツのため、フォントとline-heightに注意する

### コンテンツ方針
- 保護者目線で安心感を与えるトーン
- 押し売り感を出さない（「スクールに通うべき」ではなく「一歩踏み出す背中を押す」）
- データや根拠に基づいた説得力のある内容
- 視覚的に読みやすく、途中離脱を防ぐ構成

## ディレクトリ構成
```
/
├── index.html          # メインLP（単一ファイル構成）
├── images/             # 画像素材
├── articles/           # WordPress投稿用Markdownファイル
├── pages/              # WordPress固定ページ用HTMLファイル
├── scripts/            # デプロイスクリプト
│   └── deploy-to-wordpress.js
├── .github/workflows/  # GitHub Actions
│   └── deploy-to-wordpress.yml
├── package.json
└── *.pdf               # 参考資料PDF
```

## 重要な注意事項
- PDFファイルはiframeで埋め込み表示している（文部科学省資料、プログラミング教育の必要性資料）
- WordPress連携は articles/ と pages/ ディレクトリの変更をトリガーに自動デプロイ
- リモートリポジトリは2つ: `origin`(masakinakano-cpc) と `second`(masaki-sigoto)
